# 更新记录

本文件用于记录 Openxhh 的重要更新、更新时间和更新内容，方便用户不用翻 git log 也能看到项目变化。

记录规则：

- 时间使用本仓库提交时间。
- 新记录按时间倒序追加。
- 面向用户说明“改了什么、有什么影响”，不只写内部代码文件名。
- 更细的技术细节仍以 git commit 和代码为准。

## 如何继续维护

后续每次发布或完成一组重要修改时，在本文件顶部追加一段：

```md
## YYYY-MM-DD

| 时间 | 类型 | 内容 | 提交 |
| --- | --- | --- | --- |
| HH:mm | 新增/修复/文档/维护 | 面向用户说明这次更新的影响。 | `commit` |
```
